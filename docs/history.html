Historical Metrics Overview

his page provides detailed insights into the historical development of software quality metrics for USE. The diagrams below depict the evolution of metrics over the last 15 years, dating back to 2010. All metrics were collected only for the master branch commits. Two different build systems were identified - Ant and Maven - and metrics collection is additionally divided by those categories.

Historical Cyclic Dependencies - Ant, No Tests

Displays the historical numbers of dependency cycles in the entire core module of the Ant-version project, excluding tests. The core is defined as the packages in the src folder, excluding org.tzi.use.runtime and org.tzi.use.gui.

Historical Cyclic Dependencies - Maven, No Tests

Displays the historical numbers of dependency cycles in the entire use-core module of the Maven-version project, excluding tests.

Historical Cyclic Dependencies - Ant (With Tests)

jz65

Displays the historical numbers of dependency cycles in the entire core module of the Ant-version project, including tests. The core is defined as the packages in the src folder, excluding org.tzi.use.runtime and org.tzi.use.gui.

Historical Cyclic Dependencies - Maven (With Tests)

Displays the historical numbers of dependency cycles in the entire use-core module of the Maven-version project, including tests.

Historical Architecture Layer Violations - Ant

Extended timeline of whether any classes in the core module access any classes in the GUI module which would equal a violation of the layered architecture. Tests not included. The core in the Ant-version is defined as the packages in the src-folder, excluding those in org.tzi.use.runtime and org.tzi.use.gui. The GUI is defined as the packages in the org.tzi.use.gui package.

Historical Architecture Layer Violations - Maven

Extended timeline of whether any classes in the use-core module access any classes in the Maven use-gui module which would equal a violation of the layered architecture. Tests not included.

Historical Build Time - Maven

Long-term analysis of build times in the Maven USE-project. For each commit the build time is measured using time stamps and includes the building as well as the testing stage.

Historical Analysis Guide

The scripts for reproducing these metrics can be found in the USE-repository under docs/scripts. Please make sure you have Powershell 7+ installed for the execution.
